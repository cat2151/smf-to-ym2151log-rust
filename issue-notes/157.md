# issue issue 153 の調査の結果、console.logをみると、c1もc64もまったく同じSMFが生成されていることがわかった #157
[issues #157](https://github.com/cat2151/smf-to-ym2151log-rust/issues/157)

さらに、ライブラリ側を調査した結果、c1もc64もまったく同じSMFが生成されていることがわかった

https://github.com/cat2151/mmlabc-to-smf-rust/issues/109

ライブラリ側を修正したので、それを参照すべし

## 解決

- `mmlabc-to-smf-rust` の issue #109 で修正済み（PR #110）
  - `pass2_ast.rs` において、ノート固有の音長（c1, c64 など）が最高優先度で適用されるよう修正された
  - 修正前：chord の明示的な音長 → l コマンド → デフォルト4 の優先順位で、ノート固有の音長が無視されていた
  - 修正後：ノート固有の音長 → chord の明示的な音長 → l コマンド → デフォルト4 の優先順位になった
- 本プロジェクトのデモは GitHub Pages の URL から最新ライブラリを動的に読み込むため、修正は自動的に反映される
- デモの MML プレースホルダーを `c1c4c8c64;` に更新し、音長の違いが視覚的に確認できるようにした
